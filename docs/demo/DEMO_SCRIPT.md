# Codequake Demo Script

1. Open Codequake and show the repository URL plus PR diff inputs.
2. Click "Run IBM Bob reasoning" or paste an exported IBM Bob architecture report.
3. Narrate: "IBM Bob is reading the repository as an architecture, not as isolated files."
4. Show the Codequake Intensity changing to a major or critical architectural event.
5. Show the dependency topology lighting up with animated shockwave edges.
6. Open the Fault-Zone Map and call out the Authentication Fault Zone or Revenue Critical Fault Zone.
7. Show architecture warnings and stability test targets.
8. Close with: "Codequake uses IBM Bob's repository-wide reasoning to visualize software shockwaves before they spread."

## Key Narration

"A normal PR analyzer asks what files changed. Codequake asks where instability is forming in the architecture and how that instability can propagate. IBM Bob supplies the repository-wide reasoning. Codequake turns that reasoning into a visible fault-zone and shockwave model."
